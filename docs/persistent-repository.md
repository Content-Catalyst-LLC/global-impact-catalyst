# Persistent Repository

Global Impact Catalyst v1.2.0 adds a durable repository without changing the canonical v1.1.0 impact-contract shape or calculation rules.

## Storage model

The SQLite reference repository stores each canonical contract as a complete JSON snapshot and builds indexed projections for operational queries. The snapshot remains the source of truth. Projections make workspaces, initiatives, goals, indicators, observations, targets, and sources searchable without fragmenting the calculation record.

```text
canonical contract snapshot
        ├── workspace projection
        ├── initiative projection
        ├── goal projection
        ├── indicator projection
        ├── observation projection
        ├── target projection
        └── source projection
```

The repository also stores portfolios, portfolio memberships, draft autosaves, import receipts, restore receipts, and append-only audit records.

## Shared application service

`python/global_impact_service.py` contains use cases shared by the CLI and application wrapper:

- create and explicitly save initiatives;
- autosave drafts against a saved revision;
- list and search initiatives;
- duplicate initiatives while preserving measurement semantics;
- import v1.0.x records and v1.1.0 contracts;
- export and restore workspace bundles.

Interfaces should call this service rather than duplicate migration, validation, contract-building, or repository rules.

## Optimistic concurrency

Every stored contract and indexed entity has an integer revision. Updates may provide an expected revision. A stale update raises `OptimisticConcurrencyError` with the expected and current revisions. WordPress returns the equivalent `409 gic_revision_conflict` response.

This prevents a background autosave or an older browser tab from silently overwriting a newer saved record.

## Save states

- `draft autosave` is recoverable working state tied to a base saved revision.
- `saved` is the explicit durable contract revision.
- Saving a contract clears its autosave.
- Autosave never increments the explicit contract revision.

## Calculation integrity

Persistence stores the entire contract generated by the canonical engine. Reopening, exporting, importing, and restoring a contract do not recalculate or alter derived metrics. Editing compact authoring facts and explicitly saving creates a new contract snapshot through the same canonical engine.

# Global Impact Catalyst v1.0.1

## Repository Integrity and Cross-Runtime Parity Repair

v1.0.1 stabilizes the initial Global Impact Catalyst release before persistent workspaces and broader impact-data contracts are added.

### Release outcomes

- Python and browser engines now follow one observable contract.
- Twelve golden fixtures cover higher- and lower-is-better indicators, zero baselines, equal baseline and target values, movement away from target, target exceedance, negative and decimal values, blank evidence fields, invalid enum normalization, and optional null/zero behavior.
- The Python compatibility parser now handles recognized boolean strings explicitly and rejects malformed booleans and non-finite numbers.
- Input and output schemas are versioned and validate canonical sample data and generated records.
- The output schema fully defines metrics, quality components, enums, timestamp format, nullability, required fields, and fixed traceability steps.
- The WordPress shortcode can be rendered multiple times without duplicate field IDs.
- The demo reports validation errors accessibly and makes the quality score’s limited meaning explicit.
- Release checks enforce agreement among the manifest, plugin, schemas, changelog, shortcode, fixture suite, and release notes.

### Compatibility

The record remains a v1.0-series flat impact record. v1.0.1 adds `metrics.data_quality_components` while preserving all prior top-level fields and the shortcode `[global_impact_catalyst_demo]`.

### Boundaries

Global Impact Catalyst organizes measurements and calculations for review. It does not provide ESG assurance, SDG certification, regulatory compliance, causal proof, audit findings, or automatic truth verification.

### Validation

Run the portable release suite:

```bash
python3 -m pip install -r requirements-dev.txt
python3 scripts/smoke_test.py
node scripts/check_browser_parity.js
php -l wordpress/global-impact-catalyst-demo/global-impact-catalyst-demo.php
```

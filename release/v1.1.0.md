# Global Impact Catalyst v1.1.0

## Canonical Impact Contract and Validation Engine

v1.1.0 moves Global Impact Catalyst from a single flat measurement record to a durable, versioned impact-data contract.

### Contract model

The release creates canonical entities for workspace, initiative, goal, outcome, indicator, indicator-definition version, baseline, target, observation, source, method, population group, geography, budget record, interpretation, claim, review, revision, and publication. Each material entity receives a deterministic ID and timestamps.

Entered facts are stored under `facts`. Calculations, interpretations, and claims are stored under `derived`. Review, revision, and publication state are stored under `governance`. Provenance and validation remain first-class root sections.

### Validation engine

Validation returns structured issues rather than unstructured exceptions. Each issue includes:

- stable issue ID;
- `error`, `warning`, or `info` severity;
- contract field path;
- rule ID;
- plain-language message;
- actionable remediation.

The initial rule set checks required impact facts, measurement-unit compatibility, period ordering, indicator direction, baseline/target compatibility, source locators, method detail, and claim eligibility.

### Claim governance

The contract distinguishes descriptive observations, progress-to-target statements, comparisons, contribution statements, and causal claims. Comparison, contribution, and causal language cannot become eligible without explicit claim text and the design metadata appropriate to the claim type. Causal claims additionally require high confidence and completed review.

### Migration

The migration function converts v1.0.x flat records into the v1.1.0 structure. The original legacy record is retained unchanged under provenance, ensuring no source content is discarded during migration.

### Cross-runtime parity

Python and browser implementations produce identical canonical contracts for fifteen golden fixtures, including stable entity IDs, fingerprints, validation issue IDs, derived values, claim gates, governance records, and boundary language.

### Exit criteria

- v1.0.x records migrate without content loss.
- Invalid contracts return structured, actionable issues.
- Derived values identify their calculation engine, version, rounding rule, and formula.
- Contract field definitions, validation rules, claims, and migration behavior are documented.

### Boundaries

This release does not provide ESG assurance, SDG certification, regulatory compliance, audit conclusions, evidence authentication, causal proof, or automatic truth verification.

# Global Impact Catalyst v1.3.0

## Sources, Provenance, and Evidence Chain

v1.3.0 adds a first-party evidence repository to the persistent impact platform. Canonical source identifiers now materialize into structured source records and immutable SHA-256 versions. Users can capture evidence excerpts, register datasets, record provenance edges, and explicitly link evidence to claims as supporting, contradicting, qualifying, or contextual.

### Major capabilities

- fourth repeatable SQLite migration
- source registry and optimistic source revision control
- immutable source-version ledger and checksum verification
- evidence excerpt and quotation capture
- dataset metadata and integrity registry
- directed provenance graph
- explicit claim-evidence relationships and contradiction visibility
- evidence-chain and evidence-repository schemas and examples
- complete evidence backup and restore inside workspace bundles
- expanded CLI and authenticated WordPress evidence ledger

### Compatibility

The canonical calculation engine and contract remain v1.1.0. v1.3.0 is an additive repository and interface release. Existing v1.2.0 databases migrate in place, and existing contract saves automatically create source versions and provenance edges.

### Boundaries

Checksums establish change detection, not truth or authenticity. Evidence links record declared relationships, not adjudicated proof. Stronger claims still require appropriate methods, human review, and responsible publication controls.

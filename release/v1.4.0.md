# Global Impact Catalyst v1.4.0

## Indicator Registry, Units, Baselines, Targets, and Methods

v1.4.0 adds a first-party measurement-definition registry to the persistent, evidence-backed impact platform. Indicators can now reference governed units, immutable definition versions, explicit baseline and target models, constrained formulas, and reusable method definitions.

### Major capabilities

- fifth repeatable SQLite migration
- eighteen seeded standard units and workspace-specific custom units
- dimension-aware unit conversion and incompatibility rejection
- versioned indicator definitions with direction, unit, formula, disaggregation, and quality metadata
- safe `gic-expression-1.0` formula validation and evaluation
- point, average, rolling, benchmark, and modelled baseline computation
- absolute, range, relative, linear, step, exponential, and custom target evaluation
- reusable method definitions with input requirements, quality profiles, and limitations
- stable initiative-indicator bindings to exact definition and model versions
- automatic registry materialization from saved canonical contracts
- indicator-registry schema, example, export, and lossless workspace restore
- expanded Python service, CLI, WordPress tables, REST routes, and authenticated registry shortcode

### Compatibility

The canonical calculation engine and contract remain v1.1.0. The evidence-chain and evidence-repository formats remain v1.3.0. v1.4.0 is an additive repository, bundle, registry, and interface release. Existing v1.3.0 databases migrate in place, and existing contract saves automatically materialize registry records and bindings.

### Boundaries

Unit compatibility does not prove methodological comparability. A registered formula is a transparent calculation rule, not causal evidence. Baseline and target models are governed assumptions and trajectories, not assurance opinions or certified outcomes. Human review remains required.
